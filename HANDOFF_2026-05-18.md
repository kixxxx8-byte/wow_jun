# WJ+ Command 인수인계 메모

작성일: 2026-05-18  
프로젝트: Firebase `hokkaido-trip-c1907` / 공개 경로 `/v8/`  
현재 작업 폴더 기준: `recovered-site`

## 현재 상태

- 앱은 React + Vite + TypeScript 단일 앱입니다.
- 공개 사이트는 `https://hokkaido-trip-c1907.web.app/v8/` 입니다.
- Firebase Functions는 `asia-northeast3`에 `api`, `bnetStart`, `bnetCallback`, `bnetSync`가 배포되는 구조입니다.
- 직전 안정 배포에는 캐릭터 드롭다운 중복/`이전` 표시 정리, Battle.net 동기화 개선, 장비 조감도 1차 구현이 포함되어 있습니다.
- 현재 로컬에는 **실제 WoW 아이템 툴팁 하이브리드 구현**이 추가되어 있습니다.
- 이 툴팁 작업은 테스트/빌드/E2E까지 통과했지만, 사용자가 압축 요청으로 중단해서 **아직 라이브 배포는 완료하지 않았습니다.**

## 이번 로컬 변경 요약

- 모든 아이템 아이콘을 공통 `ItemTooltipAnchor` 경로로 연결했습니다.
- Wowhead hover 툴팁을 기본으로 붙이고, 스크립트 실패/모바일/키보드 focus 시 Battle.net 공식 데이터 fallback 카드가 뜨게 했습니다.
- 새 API 경로를 추가했습니다.
  - `GET /api/items/tooltip?region=kr&locale=ko_KR&itemId=123`
  - Functions 내부에서 Battle.net `/data/wow/item/{id}`와 `/data/wow/media/item/{id}`를 조회합니다.
  - Firestore `itemTooltips/{region-locale-itemId}`에 캐시합니다.
- `firebase.json`에 `/api/items/**` rewrite를 추가했습니다.
- 목표 아이템 일부에 실제 item id를 매핑했습니다.
  - 알게타르 수수께끼 상자: `193701`
  - 태양섬광 분광경: `252420`
  - 백금 별의 고리: `193708`
  - 공허의 맞물림: `251217`
  - 크리크의 딱정벌레 단도: `133491`
  - 찬란한 분리검: `251212`
  - Wayfinder Boots: `72849`
- `hands-dungeon` 목표는 공식 item id가 아직 확실하지 않아 `tooltipName`만 둔 상태입니다.

## 다른 컴퓨터에서 여는 법

1. 압축을 풉니다.
2. PowerShell을 열고 압축 푼 `recovered-site` 폴더로 이동합니다.

```powershell
cd C:\path\to\recovered-site
```

3. 의존성을 설치합니다.

```powershell
.\.tools\node-v24.14.0-win-x64\npm.cmd install
cd functions
..\.tools\node-v24.14.0-win-x64\npm.cmd install
cd ..
```

4. 검증합니다.

```powershell
.\.tools\node-v24.14.0-win-x64\npm.cmd test
.\.tools\node-v24.14.0-win-x64\npm.cmd run build
cd functions
..\.tools\node-v24.14.0-win-x64\npm.cmd run build
cd ..
.\.tools\node-v24.14.0-win-x64\npm.cmd run e2e
```

5. Firebase 로그인 상태를 확인합니다.

```powershell
.\.tools\node-v24.14.0-win-x64\npm.cmd run firebase:login:list
```

로그인이 안 되어 있으면:

```powershell
.\.tools\node-v24.14.0-win-x64\npm.cmd run firebase:login
```

6. 라이브 배포합니다.

```powershell
.\.tools\node-v24.14.0-win-x64\npm.cmd run deploy
```

## Secret 주의

압축본에는 API 키와 secret 파일을 넣지 않았습니다.

Cloud Secret Manager에는 이미 다음 secret이 설정되어 있어야 합니다.
- `GEMINI_API_KEY`
- `BNET_CLIENT_ID`
- `BNET_CLIENT_SECRET`

확인:

```powershell
.\.tools\node-v24.14.0-win-x64\npm.cmd run secrets:check
```

로컬 emulator 테스트를 하려면 새 컴퓨터에서 `functions/.secret.local`을 직접 만들어야 합니다. 이 파일은 압축본에 포함하지 않았습니다.

## 다음 작업 추천 순서

1. 압축본을 새 컴퓨터에서 풀고 의존성 설치.
2. `npm test`, `npm run build`, `functions build`, `npm run e2e` 확인.
3. `/api/items/tooltip?region=kr&locale=ko_KR&itemId=193701`이 JSON을 반환하는지 배포 후 확인.
4. 장비 탭에서 아이템 아이콘 hover/focus/tap 툴팁 확인.
5. 장비 조감도 디자인은 아직 “너무 크다” 피드백이 남아 있으므로, 다음 작업에서 캐릭터 중심 compact 보드로 다시 줄이는 것이 좋습니다.
